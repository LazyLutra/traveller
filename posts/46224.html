HTTP（四）

征蓬2021-03-312023-09-24

重点

首先需要明确几个内容：

  1. TCP/IP协议是由 TCP协议 和 IP协议 组成。

    在TCP/IP的四层模型中，前者在传输层，负责在两台通信的主机之间建立连接；

    后者在网络层，负责处理不同网络之间的发送的数据包，将其分片、重组，以及根据IP地址做路由。

  2. HTTP协议是建立在 TCP协议 的基础上。

    每次请求网页时，HTTP会通过TCP建立起一个到服务端的通道，当请求数据都获取完毕后，HTTP会将TCP连接断开。

  3. HTTP是无状态的短连接。

    每次向服务端发起请求都要重新建立连接，请求结束后关闭，不记录任何状态。

  4. HTTPS为什么安全

    使用SSL保证了传输安全，防止被窃听、窃取数据，可以确认网站的安全性。

  5. 为什么需要证书

    防止被“中间人”攻击，可以证明网站身份。

HTTP

报文

请求报文

HTTP的请求报文由 请求行、请求头、空行、请求内容实体 四部分组成。

  1. 请求行

    由请求方法、URL、协议版本三部分组成。使用空格隔开。

  2. 请求头

    由 Key-Value 组成，使用冒号隔开，末尾使用回车符和换行符。

    常用的请求头有：

    • User-Agent : 请求的浏览器类型。
    • Accept : 客户端可识别的响应内容列表。
    • Accept-Language : 客户端可接受的自然语言。
    • Accept-Encoding : 客户端可接受的编码格式。
    • Accept-Charset : 客户端可接受的字符集。
    • Host : 请求的主机名，允许多个域名绑定一个IP地址。
    • Connection : 连接方式（close或keeplive）。
    • Cookie : 存储在客户端的扩展字段。

  3. 空行

    最后一个请求头后是空行，告知服务端，以下内容不再是请求头内容了。

  4. 请求内容

    主要用于POST请求（GET请求内容拼接到URL中）。

    配套的请求头一般有：

    • Content-Type : 标识请求内容的类型。
    • Content-Length : 标识请求内容的长度。

响应报文

响应报文由 状态行、响应头、空行、响应内容实体 四部分组成。

  1. 状态行

    由HTTP协议版本、状态码、状态码描述三部分组成。

    常用的状态码如下：

    • 1xx : 服务器已接收到客户端的请求，客户端可以继续发送请求。
    • 2xx : 服务器已成功接收到请求并处理。
    • 3xx : 服务器要求客户端重定向。
    • 4xx : 客户端的请求有非法内容。
    • 5xx : 服务端未能正确处理客户端的请求，出现错误。

  2. 响应头

    常见的内容有：

    • Location : 服务端返回给客户端，用于重定向的新地址。

    • Server : 服务端用来处理请求的软件信息和版本信息。

    • Vary : 标识不可缓存的请求头列表。

    • Connection : 连接方式。

    • close

      • 客户端：告诉服务端，断开连接，不必等待后续的请求。
      • 服务端：连接已经关闭。

    • keeplive

      • 客户端：告诉服务端，完成本次请求的响应后，保持连接，等待后续请求。
      • 服务端：如果客户端需要保持连接，则该字段需要表明服务器需要保持连接的时间，例如 Keep-Alive: 300 （保持连接300秒）。

  3. 空行

    最后一个请求头后是空行，告知服务端，以下内容不再是请求头内容了。

  4. 响应内容

    服务端返回给客户端的文本信息。

文件分块编码传输

分块传输编码只在 HTTP 1.1 版本中提供。

HTTP的应答消息数据都是整个发送的，Content-Length 消息头表示数据的长度，标识了应答消息的结束，后续应答消息的开始。

如果使用分块传输数据，通常数据块的大小是一致的。

使用分块传输数据优点如下：

  1. 允许服务器为动态生成的内容维持HTTP持久连接。

  2. 允许服务器在分块数据最后发送消息头字段。

    这个功能在内容生成前无法获取头字段值的情况下很重要。

  3. 缩短传输时间。

HTTP长连接

  1. 定义

    客户端和服务端建立连接后不断开，进行报文的发送和接收，通讯连接一直存在。

    常用于点对点的通信。

  2. 步骤

    建立连接 —— 数据传输 …… 数据传输 —— 断开连接

  3. 常见场景

    监控系统、数据库连接（如果使用短连接频繁通信，容易造成Socket错误）等

HTTP短连接

  1. 定义

    客户端和服务端每进行一次报文的发送和接收，完毕后就断开连接。

    常用于操作频繁，一对多的通信。

  2. 步骤

    建立连接 —— 数据传输 —— 断开连接
    建立连接 —— 数据传输 —— 断开连接

  3. 常见场景

    web网站的http服务。

GET和POST的区别

  1. 传输方式

    • GET：地址栏传输，将请求参数拼接到URL后

    • POST：报文传输

  2. 传送长度

    • GET：长度受限于URL长度

    • POST：无限制

  3. 产生TCP数据包的数量

    • GET：一个

    • POST：两个

  4. 安全性

    • GET：不安全。参数拼接到URL后面，而URL是可见的

    • POST：相对GET安全性较高

  5. 支持的字符集

    • GET：ASCII字符（不支持中文），如果要传输中文参数，需要提前处理（URL encode等）

    • POST：form所在的html文件的开头处指定

HTTPS

HTTP和HTTPS的区别

  1. URL地址的开头部分不同

    • HTTP : http://

    • HTTPS : https://

  2. 数据加密和安全性

    • HTTP : 无法加密，不安全

    • HTTPS : 可以加密，安全

  3. 标准端口

    • HTTP : 80

    • HTTPS : 443

  4. CA证书

    • HTTP : 不需要证书

    • HTTPS : 需要SSL证书

cookie和session

维持一个会话的核心就是客户端的唯一标识，即 session id。

区别

  1. 保存的位置

    • cookie : 浏览器

    • session : 可以存放在服务端的文件、数据库、内存中

  2. 安全性

    • cookie : 容易进行cookie欺骗

    • session : 无限制

  3. 运行依赖

    • cookie : 无

    • session : 依赖于 session id，而session id保存于cookie中

其他

  1. Tomcat

    当使用 kill 命令关闭Tomcat进程时，session会全部丢失；如果手动关闭，则下次启动时，会重新加载到内存中。

    由此可见，在运行时，session保存在内存中。
